Exercice 1. Répresentation et manipulation d'expressions arithmétiques.

On souhaite pouvoir représenter des expressions arithmétiques avec variables par un type OCaml puis manipuler ces représentations. Des exemples d'expressions devant pouvoir être représentées suivent :

  • (2×x) + 1

  • 42mod7 + 12×2

  • (a + b) ⁄ (a − b)

  • 1 + max(x, 0)

  • sqr(abs(x − y))

  •  − x + 2

Question 1 : types

  1. Définir un type unary_operation qui permet de représenter les opérations unaires (l'opposé, la valeur absolue, le carré).

  2. Définir un type binary_operation qui permet de représenter les opérations binaires.

  3. Définir un type paramétré 'a expression pour définir des opérations arithmétiques avec variables. Il est attendu que ce type ait quatre constructeurs. Dans le constructeur qui les concernent, les variables sont représentées par des valeurs de type 'a.

  4. En utilisant les constructeurs de ces types, définir des valeurs qui représentent les expressions arithmétiques ci-dessus.

Question 2 : évaluation des expressions

  1. Ecrire une fonction closed: 'a expression -> bool qui renvoie true si et seulement si l'expression est close, c'est-à-dire qu'elle ne contient pas de variables.

  2. Ecrire une fonction all_variables: 'a expression -> 'a list qui renvoie la liste de toutes les variables présentes dans l'expression. Une variable présente plusieurs fois dans l'expression sera présente plusieurs fois dans la liste.

  3. Ecrire une fonction variables: 'a expression -> 'a list qui renvoie la liste de toutes les variables présentes dans l'expression, mais sans doublons dans la liste.

Question 3 : analyse des expressions

  1. Ecrire une fonction get_unary: unary_operation -> int -> int qui à la représentation d'une opération unaire fait correspondre son implantation en OCaml.

  2. Ecrire une fonction get_binary: binary_operation -> int -> int -> int qui fait de même pour les opérations binaires.

  3. Ecrire une fonction evaluate_closed: 'a expression -> int qui évalue des expressions closes (c'est-à-dire qui ne contiennent pas de variables). Si l'expression donnée en argument n'est pas close, la fonction lève une exception appropriée.

Exercice 2. Bonus

Cet exercice est la suite de l'exercice 1.

Question 1

Ecrire une fonction eval_with_store : ('a -> 'b -> int) -> 'b -> int option -> 'a expression -> int qui permet d'évaluer des expressions avec variables.

Dans un appel eval_with_store get store default_value expression, les valeurs pour ces variables sont stockées dans une structure de données de dictionnaire store et le premier argument get est une fonction qui étant donnés une telle structure de données et une variable : soit retourne l'entier associé à cette variable, soit lance l'exception Not_found. Le troisième argument est une valeur par défaut optionnelle. Si jamais une variable n'est pas présente dans le dictionnaire et si une valeur par défaut est donnée, cette valeur est utilisée. Si une variable n'est pas dans le dictionnaire et que la valeur par défaut est None, la fonction eval_with_store lève une exception.

Question 2

En utilisant, eval_with_store écrire une fonction evaluate : ('a * int) list -> 'a expression qui prend une liste d'association comme dictionnaire pour les variables.

Si le premier exemple d'expression de l'exercice 1 est implantée par une expression ex1 de type char expression et que la variable x est représentée par le caractère 'x', la boucle interactive doit répondre par exemple :

1    # evaluate [('x',1)] ex1;;
2    - : int = 3
